# TryHackMe - Operating System Security

## Introduction

Operating System Security refers to the mechanisms and practices used to protect an operating system from unauthorized access, misuse, or attacks.

Since operating systems manage hardware, applications, users, and system resources, they are a critical component of system security.

Proper operating system security helps protect:

- system files
- user data
- running processes
- network communications

---

## Why Operating System Security Matters

Operating systems are often the **primary target for attackers**.  
If an attacker compromises the operating system, they can potentially gain full control over the system.

Security mechanisms help prevent:

- unauthorized access
- privilege escalation
- malware infections
- system manipulation

A secure operating system ensures **confidentiality, integrity, and availability (CIA triad)** of system resources.

---

## User Accounts and Permissions

Operating systems use **user accounts and permission systems** to control access to files and resources.

Different types of users typically exist:

- **Standard users**
- **Administrators**
- **System accounts**

Permissions define what users can do, such as:

- reading files
- modifying files
- executing programs
- managing system settings

Proper permission management is essential to prevent unauthorized actions.

---

## Authentication

Authentication verifies the identity of a user before granting access to the system.

Common authentication methods include:

- passwords
- PIN codes
- biometric authentication
- multi-factor authentication (MFA)

Strong authentication mechanisms help prevent unauthorized access.

---

## Password Security

Passwords remain one of the most common authentication methods.

Good password practices include:

- using long and complex passwords
- avoiding reused passwords
- storing passwords securely (hashed and salted)
- enabling multi-factor authentication

Weak passwords are a common entry point for attackers.

---

## System Updates and Patching

Operating systems regularly receive **security updates and patches**.

These updates fix:

- software vulnerabilities
- security flaws
- system bugs

Keeping systems updated is critical because attackers often exploit **known vulnerabilities** in outdated systems.

---

## Malware Protection

Operating systems use several mechanisms to protect against malware:

- antivirus software
- security policies
- application restrictions
- system monitoring

Malware can include:

- viruses
- trojans
- ransomware
- spyware

Proper protection mechanisms help detect and prevent malicious activity.

---

## Firewalls

A firewall monitors and controls network traffic entering or leaving the system.

Firewalls help protect systems by:

- blocking unauthorized connections
- filtering suspicious traffic
- enforcing network security rules

Both **host-based firewalls** and **network firewalls** are commonly used.

---

## Principle of Least Privilege

The **Principle of Least Privilege (PoLP)** states that users and programs should only have the permissions necessary to perform their tasks.

Benefits include:

- reducing attack surfaces
- limiting the damage of compromised accounts
- preventing accidental system changes

This principle is widely used in secure system design.

---

## Key Takeaways

- Operating systems are critical components that must be secured.
- Security mechanisms include authentication, permissions, and system updates.
- Malware protection and firewalls help defend against attacks.
- Proper user management and the principle of least privilege improve security.
- Maintaining system security is essential in cybersecurity environments.

---

## Conclusion

Operating System Security focuses on protecting the system and its resources from unauthorized access and malicious activity.

Understanding these security mechanisms is essential for anyone working in **cybersecurity, system administration, or IT infrastructure protection**.
